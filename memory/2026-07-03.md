# RNGD LLM 연구 — 2026-07-03 세션 기록

## 0. 오늘의 큰 방향

오늘은 기존 attention/TP=8 결과를 바로 확장하기보다, 문제를 더 작은 단위로 쪼개기로 했다.

핵심 질문:

> Single kernel latency로 Double을 설명할 수 있는가?
> Double 결과로 Triple을 설명할 수 있는가?
> Triple/Double 결과로 Quad를 설명할 수 있는가?

즉, 합성(fusion/composition) 효과가 **국소적이고 재귀적으로 예측 가능한지**를 먼저 보려는 방향이다.

TP=8은 communication/synchronization latency가 섞일 수 있으므로, 이번 controlled ladder 실험은 **TP=1**로 고정했다.

## 1. 프로파일러 경로 재확인

현재 사용 중인 것은 별도 CLI/UI profiler가 아니라 `furiosa.torch`의 저수준 compiled-module profiling path다.

경로:

```text
torch.nn.Module
-> torch.export.export(...)
-> CompileModule.from_exported(...)
-> cm.generate_profiles(device)
-> cm(..., profiles=profiles, device=device)
-> cm.edf.npu_node.build_tuc_profile_spans(...)
-> span parsing
```

주요 span:

- `Task`: device task latency. 현재 주 latency metric.
- `Renegade::TuExec`: compute execution span.
- `DMA`: data movement span.
- `Renegade::StoTrf`: 큰 attention chain에서 spill/restream처럼 보이는 span.
- `Renegade::ParallelCopy`, `Cluster`: TP=8에서 communication/synchronization 힌트로 관찰됨.

주의:

- 이 profiler는 "PyTorch op 하나 = kernel row 하나"를 직접 주는 방식이 아니다.
- compiled block의 device timeline span을 주므로, fusion은 `L(chain)`과 `sum(L(single components))` 비교로 추론한다.

## 2. TP=8 attention chain 결과 재정리

이전에 보여준 single-kernel baseline은 모두 **TP=8에서 측정된 값**이었다.

TP=8에서 측정된 attention-chain 결과:

### MatMul -> Softmax -> MatMul

```text
S=128:  chain 21.24 us, single sum 44.44 us, gap -23.19 us
S=512:  chain 130.78 us, single sum 217.22 us, gap -86.44 us
S=2048: chain 1234.55 us, single sum 2034.20 us, gap -799.66 us
```

### Softmax -> MatMul -> MatMul -> Add

```text
S=128:  chain 132.84 us, single sum 168.03 us, gap -35.19 us
S=512:  chain 406.10 us, single sum 487.27 us, gap -81.16 us
S=2048: chain 1694.62 us, single sum 2281.40 us, gap -586.78 us
```

현재 해석:

- TP=8에서는 측정된 attention chain들이 모두 isolated single sum보다 빠르다.
- 다만 TP=8은 communication/synchronization latency까지 섞일 수 있으므로, fusion 자체를 이해하려면 TP=1 controlled ladder가 먼저 필요하다.

## 3. Controlled primitive ladder 실험 설계

목표:

```text
Single Kernel -> Double Kernel -> Triple Kernel -> Quad Kernel
```

이 순서로 합성 효과가 어떻게 누적되는지 본다.

Primitive vocabulary:

- `M`: MatMul
- `A`: Add
- `S`: Softmax

초기 shape:

```text
B = 1
T = 512
D = 1024
dtype = bfloat16
input = [1, 512, 1024]
```

각 primitive는 shape-preserving이다.

```text
M(x) = x @ W, W=[1024,1024] -> [1,512,1024]
A(x) = x + residual -> [1,512,1024]
S(x) = softmax(x, dim=-1) -> [1,512,1024]
```

조합 수:

```text
Single: 3
Double: 9
Triple: 27
Quad: 81
Total: 120
```

## 5. Single/Double/Triple/Quad 결과 상태

결과 파일:

```text
profiling/primitive_ladder/primitive_ladder_tp1_len4_results.csv
profiling/primitive_ladder/logs/primitive_ladder_tp1_len4_run.log
```

전체 결과 요약:

```text
Length   count   median chain   median single-sum   median gap
Single     3       17.20 us        17.20 us             0.00 us
Double     9       65.41 us        81.06 us           -11.94 us
Triple    27       72.77 us        98.26 us           -23.06 us
Quad      81       85.95 us       115.45 us           -32.89 us
```

Gap range:

```text
Double: min -15.65 us, median -11.94 us, max -8.53 us
Triple: min -30.54 us, median -23.06 us, max -13.58 us
Quad:   min -45.72 us, median -32.89 us, max -19.20 us
```

현재 관찰:

- 이 medium TP=1 shape에서는 anti-fusion이 보이지 않았다.
- 모든 Double/Triple/Quad chain이 isolated single sum보다 빠르다.
- gap은 chain length가 길어질수록 대략 누적된다.
- latency 자체는 MatMul 개수에 크게 좌우된다.
- 하지만 fusion gap은 MatMul 개수보다 chain length와 boundary 수에 더 안정적으로 반응하는 것처럼 보인다.

## 6. 해석은 아직 보류

주현은 Quad까지 나온 결과를 보고, 어떻게 모델링할지 더 생각하기로 했다.

현재 가능한 해석 후보:

1. **Boundary benefit 모델**
   - 각 composition boundary마다 대략 10~12 us 정도 benefit이 누적되는 형태.

2. **Pair-gap 모델**
   - Double gap으로 Triple을 어느 정도 설명할 수 있음.
   - Triple pair-gap model residual median은 거의 0에 가까웠음.

3. **Longer-chain extra fusion**
   - Quad는 Double+Double 또는 Triple+Single로 예측해도 추가로 약 10 us 정도 더 빠른 경향이 있음.
   - 즉 boundary를 하나 더 붙일 때 추가 fusion/overlap이 생기는 듯함.

아직 확정 결론은 내리지 않는다.
